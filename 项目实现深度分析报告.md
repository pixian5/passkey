## 1. apps/copilot-53-flutter

**✅ 可用的实际项目**

- **平台识别**：自动显示当前运行平台（通过 `Platform.operatingSystem`）

#### 技术实现细节

**架构设计**：

- 右侧面板：账号列表（ListTile 展示）
- AppBar：包含生成演示数据的魔法棒按钮

**关键代码路径**：

## 2. apps/copilot-53-tauri

### 项目状态
**✅ 可用的生产级项目**

### 项目概述
基于 Tauri 2 的跨平台桌面应用，采用 Rust 后端 + HTML/CSS/JavaScript 前端架构。

### 核心功能实现

#### 已实现的完整功能
1. **设备名管理**
   - 设置与持久化
   - 默认值：`Copilot53Desktop`

2. **账号完整生命周期管理**
   - 创建账号（支持多站点、用户名、密码、TOTP、恢复码、备注）
   - 编辑账号
   - 软删除（移入回收站）
   - 恢复账号
   - 彻底删除

3. **别名域自动合并**
   - 使用并查集（Union-Find）算法
   - 站点集合有交集时自动合并
   - 实现路径：`src-tauri/src/main.rs:304-358`

4. **回收站机制**
   - 软删除标记 `is_deleted` 字段
   - 记录删除时间 `deleted_at_ms`
   - 支持恢复和彻底删除

5. **数据持久化**
   - SQLite 数据库（WAL 模式）
   - 位置：`<app_local_data_dir>/pass-copilot-53-tauri.db`
   - KV 存储结构

6. **CSV 导出**
   - 导出所有非删除账号
   - 时间格式：`yy-M-d H:m:s`（与 macOS 版对齐）
   - 导出路径：`<app_local_data_dir>/pass-export-*.csv`

7. **演示数据生成**
   - 一键生成三个示例账号
   - GitHub、Google、Example 域名

#### 技术架构

**前端（JavaScript）**：
- 原生 JavaScript（无框架）
- DOM 操作
- Tauri API 调用
- 文件：`src/main.js`（181行）

**后端（Rust）**：
- Tauri 2 框架
- rusqlite（SQLite）
- serde（JSON 序列化）
- chrono（时间处理）
- uuid（ID 生成）
- 文件：`src-tauri/src/main.rs`（482行）

**数据模型**：
```rust
struct PasswordAccount {
    id: String,
    account_id: String,
    canonical_site: String,
    sites: Vec<String>,
    username: String,
    password: String,
    totp_secret: String,
    recovery_codes: String,
    note: String,
    created_at_ms: i64,
    updated_at_ms: i64,
    created_device_name: String,
    last_operated_device_name: String,
    is_deleted: bool,
    deleted_at_ms: Option<i64>,
}
```

#### 核心算法分析

**别名域自动合并算法**（`sync_alias_sites`）：
1. 构建站点到账号索引的映射
2. 使用 DFS 遍历连通分量
3. 合并所有相关账号的站点集合
4. 按字母排序并更新所有账号
- 时间复杂度：O(N + M)，N 为账号数，M 为站点总数
- 空间复杂度：O(M)

**数据库架构**：
```sql
CREATE TABLE IF NOT EXISTS kv (
    key TEXT PRIMARY KEY NOT NULL,
    value TEXT NOT NULL
);
PRAGMA journal_mode=WAL;
PRAGMA synchronous=NORMAL;
```

#### Tauri Command API

实现了 8 个 Tauri 命令：
1. `health_check()` - 健康检查
2. `get_app_state()` - 获取应用状态
3. `set_device_name()` - 设置设备名
4. `create_account()` - 创建账号
5. `update_account()` - 更新账号
6. `soft_delete_account()` - 软删除
7. `restore_account()` - 恢复账号
8. `hard_delete_account()` - 彻底删除
9. `generate_demo_accounts()` - 生成演示数据
10. `export_csv()` - 导出 CSV

## 附录：技术债务与风险

### 1. 数据安全
所有项目的密码都是明文存储（JSON 或 SQLite），建议：
- 实现加密存储
- 使用系统密钥链（Windows Credential Manager, Linux Secret Service）
- 主密码保护

### 2. 并发控制
- JSON 文件存储没有并发控制
- SQLite 使用 WAL 模式，有基本并发支持
- 建议实现文件锁或进程锁

### 3. 错误处理
- Tauri 项目有较好的错误处理
- Flutter 项目错误处理相对简单
- 建议统一错误处理策略

### 4. 测试覆盖
- 大部分项目缺少单元测试
- 建议增加测试覆盖率
- 特别是核心算法（别名合并）需要测试

### 5. 依赖管理
- Flutter 项目依赖较少，风险较低
- Tauri 项目依赖较多，需要定期更新
- 建议使用 Dependabot 或类似工具
