# 01 目标与边界

## 1. 目标
- 开发 Windows、Ubuntu 桌面版本，功能尽可能复刻 macOS 版核心能力：
  - 设备名称设置与持久化
  - 账号新增/编辑/删除/恢复
  - 站点别名同步
  - CSV 导入导出
  - 同步协议（后续阶段）
  - 统一时间格式策略

## 2. 明确排除（macOS 专有）
- AutoFill Extension
- Keychain 原生能力
- macOS 专有菜单栏与系统集成扩展

## 3. 非功能要求
- 业务规则统一下沉至 Rust Core，Flutter 主要负责展示与交互。
- 跨平台一致性优先，性能与安全性次优先后续补齐。
- 以开发测试版为前提，不强制兼容旧原型数据结构（可迁移即可）。

## 4. 完成定义（Definition of Done）
- Windows 与 Ubuntu 均可编译和启动桌面程序。
- 核心能力（账号、别名、回收站、CSV）在两端行为一致。
- Rust FFI 提供稳定 API，Flutter 端不再重复关键业务决策逻辑。
- 关键路径具备自动化测试与人工验收记录。
