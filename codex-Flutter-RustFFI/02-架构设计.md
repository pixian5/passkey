# 02 架构设计

## 1. 总体分层
- **UI 层（Flutter）**：页面、交互、状态绑定。
- **应用层（Flutter）**：用例编排、错误映射、会话状态。
- **桥接层（Dart FFI）**：将 Dart 类型映射到 C ABI。
- **核心层（Rust）**：领域模型、冲突合并、存储、协议。
- **基础设施（Rust）**：SQLite/文件系统/加密（后续）。

## 2. 目录建议
- `apps/app_flutter/lib/features/*`：按功能拆分。
- `apps/app_flutter/lib/ffi/*`：动态库加载、符号绑定、类型转换。
- `core/pass_core/crates/ffi`：统一导出 C ABI。
- `core/pass_core/crates/{domain,merge,storage,transport,csvio}`：纯核心逻辑。

## 3. 数据流
1. Flutter UI 发起用例（例如创建账号）。
2. 应用层构造请求 DTO，进入 FFI。
3. Rust Core 执行规则与持久化，返回结构化结果。
4. Flutter 更新状态并反馈 UI。

## 4. 错误模型
- Rust 统一错误码：`OK / VALIDATION / NOT_FOUND / CONFLICT / IO / INTERNAL`。
- Dart 侧统一异常适配：`PassException(code, message, detail)`。

## 5. 线程模型（建议）
- FFI 调用默认同步 + isolate 包装，避免阻塞主线程。
- 对耗时任务（导入导出、全量合并）采用后台 isolate。

## 6. 安全策略（阶段化）
- 第一阶段：明文开发模式（仅用于快速验证）。
- 第二阶段：敏感字段加密（Rust 侧统一处理）。
- 第三阶段：密钥管理按平台适配（Windows DPAPI / Linux Secret Service 等）。
